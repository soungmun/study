2. 공통 동작 모델

2-1. 인증 게이트 — RequireAuth

화면 진입 시 fetch('/api/auth/me', credentials: 'include') 호출
  → 200 OK → state='authed' → children 렌더
  → 401 → state='guest' → "🔒 로그인이 필요해요" 카드 표시
또한 window 의 'auth-changed' 커스텀 이벤트를 구독해서 로그인/로그아웃 직후 자동 재검증.

2-2. 인증 이벤트 흐름

  • AuthBar 가 로그인/회원가입/소셜 콜백 후 사용자 정보를 다시 받아오고 window.dispatchEvent(new Event('auth-changed')) 발행.
  • 같은 페이지의 다른 컴포넌트(RequireAuth, AuthBar)가 이 이벤트로 자동 재패치.
  • 토큰을 직접 들고 다니지 않는다. 모든 요청에 credentials: 'include' 로 세션 쿠키만 동봉.

2-3. API 호출 패턴

fetch(`${API}/path`, {
  method: 'POST',
  credentials: 'include',                  // ← 세션 쿠키 전달
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(payload),
})
.then(async (r) => {
  if (!r.ok) {
    const text = await r.text();
    try { throw new Error(JSON.parse(text).message || text); }
    catch { throw new Error(text || `HTTP ${r.status}`); }
  }
  return r.json();
});

2-4. 라우팅 / 화면 상태 보존

  • useNavigate + useParams 로 SPA 전환.
  • NoticeList 는 검색조건(타입/키워드/페이지)을 sessionStorage 에 저장 → 상세에서 뒤로 와도 동일 상태 복원.
  • NoticeDetail 은 useRef 로 마운트당 1회만 /views POST 호출(StrictMode 중복 방지).

3. 화면별 사용자 흐름

① 헤더 / 인증 — AuthBar 컴포넌트

[로그인 전]
  [로그인] [회원가입] [카카오] [네이버] [구글]
[로그인 후]
  👤 닉네임 / 이메일   [수정] [로그아웃]

(a) 로컬 회원가입 — 이메일 인증 선행

사용자가 [회원가입] 클릭 → 팝오버 폼 표시
→ 이메일 입력 후 [인증번호 받기] 클릭
  ↳ POST /api/auth/email/send-code → 60초 쿨다운 + "발송했어요" 안내
→ 메일에서 6자리 코드 확인 후 입력 → [확인]
  ↳ POST /api/auth/email/verify-code → ✅ 인증 완료
→ 아이디·비밀번호·(닉네임)·(공지수신동의) 입력 후 [가입하고 시작하기]
  ↳ POST /api/auth/signup → 성공 시 user 상태 갱신 → 폼 닫기 → auth-changed 이벤트 발행.
→ RequireAuth 가 자동 재검증 → 본문이 인증 화면으로 전환.

(b) 로컬 로그인

[로그인] 클릭 → 아이디/비밀번호 입력 → POST /api/auth/login
→ 실패 시 폼 안에 빨간 에러 메시지 표시.
→ 성공 시 setUser(data) + auth-changed 이벤트.

(c) 소셜 로그인 — 카카오 / 네이버 / 구글

버튼 클릭 시 window.location.href = '/api/auth/{provider}/login?returnTo=...' 로 전체 페이지 이동.
→ 백엔드가 공급자 인증 페이지로 redirect → 사용자 동의 → 공급자가 백엔드 /callback 호출 → 백엔드가 세션 부여 후 returnTo (= {origin}/)로 redirect.
→ 돌아왔을 때 AuthBar 가 자동으로 /api/auth/me 를 다시 호출해 로그인 상태로 표시.
실패 시 쿼리스트링 ?auth_error=...&auth_error_description=... 가 붙어 옴 → 상단에 빨간 알약(pill) UI.

(d) 로그아웃

[로그아웃] 클릭 → POST /api/auth/logout → setUser(null) + auth-changed → 모든 보호된 화면이 게스트로 전환.

② 공지사항 목록 / 상세 / 작성 / 수정

NoticeList (/)

마운트 시 / 검색·페이지 변경 시 GET /api/notices?page&size=10&type&keyword
→ Page<NoticeListItem> 수신 → 표 + 페이지네이션 렌더.
헤더에 [+ 글쓰기] → /notices/new. 행 클릭 → /notices/{id}.
검색조건은 sessionStorage 에 보존 → 상세에서 뒤로 와도 그대로.

NoticeDetail (/notices/:id)

마운트 시 POST /api/notices/{id}/views (조회수 +1 + 상세 반환, 1회만)
→ 제목/작성자/조회수/내용/❤️ 좋아요/💬 댓글 수 표시.

[❤️ 좋아요] → POST /api/notices/{id}/like
  → {liked, count} 수신 → 즉시 UI 갱신.
  → 401 이면 "로그인이 필요합니다" alert.

[좋아요한 사람 보기] 토글 → GET /api/notices/{id}/likes → 목록 모달/리스트로 펼침.

[수정] → /notices/{id}/edit  |  [삭제] → confirm 후 DELETE /api/notices/{id} → 성공 시 목록으로 navigate.

NoticeCreate (/notices/new) & NoticeEdit (/notices/:id/edit)

- Create: 제목/내용 입력 → POST /api/notices → 성공 시 목록으로 이동.
- Edit: 마운트 시 기존 글을 GET → 폼 채움 → 저장 시 PUT /api/notices/{id} → 상세로 이동.
- 검증 오류는 백엔드의 GlobalExceptionHandler 가 줄바꿈 합쳐 400 메시지로 반환 → 폼 하단에 표시.

③ 댓글 — CommentSection

상세 페이지 하단에 임베드.
마운트 시 GET /api/notices/{id}/comments → 리스트 렌더.
새 댓글: POST /api/notices/{id}/comments (비로그인 401 → 안내).
수정: PUT /api/comments/{commentId} · 본인 아니면 403.
삭제: DELETE /api/comments/{commentId} · 본인 아니면 403.
댓글 좋아요: POST /api/comments/{commentId}/like.

④ 책 검색 — BookSearch (/books)

검색어 입력 후 submit → GET /api/books/search?query&sort&target&page&size
→ 백엔드가 카카오 도서 API 프록시 → 결과 카드 그리드 렌더.

⑤ 지도 / 주변 맛집 — MapSearch (/map)

Kakao Maps JS SDK 를 직접 사용합니다. SDK 키워드 검색 결과 + 백엔드 카테고리 API 결과를 함께 사용합니다.

초기화
  → window.kakao.maps.load() 로 SDK 준비
  → 기본 중심: 부평역 (37.4892775, 126.7246513)
  → GET /api/weather?lat&lng, GET /api/air?lat&lng 로 좌측 패널에 날씨/미세먼지 표시

키워드 검색
  → GET /api/places/search?query&page&size=15 (백엔드가 카카오 키워드 검색 프록시)
  → 결과 리스트 + 지도 마커 + 페이지네이션
  → 항목 클릭 시 InfoWindow 열기, 지도 중심 이동.

주변 맛집 (가까운 음식점)
  → GET /api/places/nearby?lat&lng&category=FD6&radius=800&size=15
  → 백엔드: 카테고리 API 시도 → 쿼터 초과/실패 시 키워드 검색으로 폴백
  → 빨간 🍽️ 마커로 별도 표시. 응답이 비어도 빈 결과로 처리(에러 토스트 X).

지도 이동/클릭
  → 빈 곳 클릭 시 좌표에서 Places.coord2RegionCode 등으로 가장 가까운 POI 추정 → 커스텀 핀 + 주소 fallback 표시.

⑥ 주식 — StockTopList (/stocks)

탭 전환에 따라 GET /api/stocks/top-gainers?limit=10 또는 GET /api/stocks/top-losers?limit=10
→ Quote 리스트 렌더 (코드/이름/등락률/현재가 등).

⑦ 카카오페이 결제

KakaoPay (/pay)

프리셋(☕ 4500원, 🍱 9000원, 🎁 100원) 또는 직접 입력.
[카카오페이로 결제하기] → POST /api/payments/kakao/ready → 백엔드가 카카오페이 ready 호출 후 nextRedirectPcUrl 등 반환
→ 프론트가 window.location.href 로 카카오페이 결제창 이동.
카카오페이 결제 완료 시 카카오가 백엔드 /api/payments/kakao/approve 콜백을 호출하고, 백엔드가 다시 프론트의 /pay/success 또는 /pay/fail 로 302 redirect.

KakaoPaySuccess (/pay/success) · KakaoPayFail (/pay/fail)

URL 쿼리(orderId, amount, itemName 또는 reason)를 읽어 결제 결과 화면 렌더.
[결제 내역 보기] 링크로 /pay/history 로 이동.

KakaoPayHistory (/pay/history)

마운트 시 GET /api/payments/kakao/my → 본인의 Payment 리스트(READY/APPROVED/CANCELED/FAILED)를 카드/표로 렌더.

⑧ 비밀번호 찾기 · 재설정

/forgot — PasswordForgot
이메일 입력 → POST /api/auth/password/forgot
→ 가입 여부와 무관하게 동일 메시지("등록되어 있다면 발송했습니다") 표시(이메일 enum 방지).

/reset?token=... — PasswordReset
쿼리에서 토큰 추출 + 새 비밀번호 입력 → POST /api/auth/password/reset
→ 성공 시 "비밀번호가 변경되었습니다" 안내 → 로그인 화면 유도.

⑨ 회원정보 수정 — ProfileEdit (/me/edit)

마운트 시 /api/auth/me 로 현재 정보 로딩.
닉네임/이메일/비밀번호/알림수신 변경 후 → PUT /api/auth/me
→ 결과 코드에 따라 표시: 소셜 전용은 이메일/비밀번호 변경 차단, 현재 비밀번호 불일치 시 "현재 비밀번호가 일치하지 않습니다" 등.
비밀번호 변경 성공 시 백엔드가 EmailService.sendPasswordChanged 발송.

⑩ 회원 목록 — UserList (/users)

GET /api/users (로그인 필요)
→ UserListItem 배열 → 닉네임/아이디/이메일/가입일 표시.
비로그인 시 백엔드가 401 → "로그인이 필요합니다" 메시지 표시.

⑪ 관리자 공지 메일 — AdminBroadcast (/admin/broadcast)

마운트 시 GET /api/admin/me → {admin, subscribers} 수신.
비관리자면 안내만 표시. 관리자(app.admin.username=test11)면:
- 제목/본문(HTML) 입력 후 [발송] → POST /api/admin/broadcast
- [지금 일일 메일 보내기(force)] → POST /api/admin/broadcast/daily-now?force=true
→ 결과(발송수/수신자수) 표시.
